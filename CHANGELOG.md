# سجل التغييرات (Changelog)

يحتوي هذا الملف على جميع التغييرات والإضافات والإصلاحات البارزة لمشروع Amanteech Bot.

## [6.0.0] - الإنتاج الاحترافي SaaS
### 🚀 الإضافات (Added)
- **نظام Honeypot المتقدم**: إضافة أوامر فخ مثل `/secret` و `/admin_panel` لحظر المتسللين تلقائياً.
- **الفحص المتقدم للملفات/الصور/QR**: دعم فحص بصمات الملفات (MD5/SHA256) عبر VirusTotal، وتحليل الصور لكشف التزوير، وفك تشفير وفحص روابط الـ QR.
- **لوحة تحكم Admin احترافية**: واجهة إدارة شاملة تعرض الإحصائيات، الإيرادات المقدرة، وإحصائيات Honeypot مع نظام أزرار Inline.
- **نظام تتبع النقاط**: تفعيل نظام نقاط الخطر (Risk Points) للمجموعات مع إجراءات تلقائية (حظر/إسكات) بناءً على السلوك.
- **تتبع Logs موحد (Structured Logging)**: أضفنا استخراج سجلات الأحداث بصيغة JSON مع تتبع تلقائي للـ `user_id`، وذلك في `utils/logger.py`.
- **سيرفر فحص الصحة و Webhooks**: تم دمج سيرفر ميني عبر `aiohttp` لإرجاع نقاط تحقق الـ Health checks على `/health` واستقبال إشعارات `Stripe Webhooks` بشكل آمن بالاعتماد على HMAC، وذلك داخل `bot.py`.
- **نظام تخزين مؤقت للذاكرة (Caching)**: إضافة تقنية `LRU` لحفظ نتائج المسح وتفادي الهدر للـ APIs وتقليل الثواني الضائعة عند فحص الملفات/الروابط المشابهة.
- **نظام AI الذكي والمحكوم**: الـ System Prompt مخصص حصراً للتجاوب بمهنية وفي مجالات السايبرسكيورتي مع قيود لا تتجاوز 300 كلمة باستخدام `httpx`.

### 🪲 الإصلاحات (Fixed)
- ضمان وجود `db.set_user_state(None)` في بنية `try/finally` لكل المعالجات منعاً لتجمّد البوت في حالة خطأ الاتصال.
- معالجات الأخطاء باتت تنصح المستخدم وتقترح حلاً مباشراً بالعربية بدل إرجاع رسائل تقنية معقدة.
- تجميع عمليات طلبات الشبكة العشوائية ضمن منصة اتصال مركزية واحدة `_async_request()` لتقليل التشتت البرمجي.

## [5.1.0] - 2026-04-18
### 🚀 الإضافات (Added)
- **نظام Rate Limiting**: تمت إضافة قيد لطلبات المستخدمين (10 طلبات في الدقيقة كحد أقصى) لحماية البوت من السلوك العشوائي والإغراق (Spam) داخل `bot.py`.
- **نظام Caching**: تم تفعيل الذاكرة المؤقتة (LRU Dictionary) لنتائج روابط `scan_url` لتقليل الاستهلاكات غير الضرورية لواجهة VirusTotal و SafeBrowsing.
- **إعادة المحاولة التلقائية (Retry Mechanism)**: تفعيل خاصية تنفيذ 3 محاولات عند فشل أي استعلام برمجي لـ Google Safe Browsing.
- **أوامر الإشراف والإيرادات (`/admin`)**: لوحة تحكم سريعة لعرض عدد المستخدمين، الفحوصات، مشتركي الباقات، والإيرادات الشهرية التقريبية في ملف مخصص `admin.py`.
- **تحسين قاعدة البيانات (Performance Enhancement)**:
  - تفعيل `PRAGMA WAL` و `PRAGMA synchronous = NORMAL` للتعامل المتزامن السريع مع SQLite.
  - بناء فهارس (`Indexes`) على حقول `user_id` و `state` و `created_at` لتسريع عمليات القراءة بمقدار `O(1)`.
- **دالة تنظيف السجلات (`cleanup_old_data`)**: آلية مسح الاستعلامات التي تجاوزت 90 يوماً لضمان عدم تضخم حجم ملف `amanteech.db`.

### 🪲 الإصلاحات (Fixed)
- ضمان وجود المعالج الأمني `can_proceed()` في كافة مسارات البوت (الروابط، الصور، الذكاء الاصطناعي) لتفعيل الاستهلاك الصحيح للباقة.
- حصر قالب الذكاء الاصطناعي الخاص بـ "Aman" ضمن القيود الأمنية المعتمدة (لا إجابات خارج إطار التكنولوجيا، 300 كلمة كحد أقصى).
- حزمة API `Stripe` مفصولة بنظامها الخاص في ملف `payments.py` الذي يتيح أمر الترقية الآمن بدون توقف البوت.

## [5.0.0] - 2026-03-10
### 🚀 الإضافات (Added)
- إضافة نظام مصيدة المخترقين `Honeypot` للمجموعات وتتبع النقاط للمستخدمين.
- حظر تلقائي للمستخدم الذي يتخطى 80 نقطة خطر.
- أوامر خفية `secret` لحظر المتطفلين بشكل آلي فور استخدامها.
- أرشفة إشعارات الأمن السيبراني السعودية `CERT-SA` و `NCA` وإرسالها للمشتركين كل 6 ساعات.
